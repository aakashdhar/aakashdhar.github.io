Hugging Face

EMO: Pretraining mixture of experts for emergent modularity

JUST NOW

AllenAI's EMO trains a 1B-active/14B-total MoE so experts specialize in semantic domains — health, code, math — rather than surface patterns like prepositions. The trick is a document-boundary constraint during pretraining: all tokens in a document share the same expert pool, pushing coherent groups to form without hand-labeled domain data. In practice, you can run just 12.5% of the experts for a given task and lose only ~3% benchmark performance, versus near-random degradation on a standard MoE.

⚡ THIS WEEK'S PATTERN

Gate every agent write behind an explicit approval step

Vibe's remote agents make every tool call visible and require approval before any write. That pattern belongs in your own agent code too — reads are cheap, writes are permanent. Wrap destructive operations in a classification check before execution, not after.

type AgentAction = {
  type: 'read' | 'write' | 'delete';
  target: string;
  payload?: unknown;
};

async function execute(action: AgentAction, approve: (a: AgentAction) => Promise<boolean>): Promise<void> {
  if (action.type !== 'read') {
    const ok = await approve(action);
    if (!ok) throw new Error(`Action blocked: ${action.type} on ${action.target}`);
  }
  await runAction(action);
}
