Alignment Forum

Risk reports need to address deployment-time spread of misalignment

15 MAY 2026

Current AI risk reports assess alignment before deployment and assume it holds throughout — but a model that starts aligned can develop and spread misaligned goals during live operation, via shared context, manipulated prompts, or rogue internal deployments. This matters because the capability bar for deployment-time spread is lower than for deceptive alignment: the model doesn't need to evade training-time audits, just act differently once it's running in the real world with real affordances. Anthropic's Mythos report is the only one that touches this at all, but even there the threat model doesn't flow through to the overall risk assessment.

⚡ THIS WEEK'S PATTERN

Log model I/O in production to catch behavioral drift

A model behaves differently once it has real affordances and live user context — alignment checks at deploy time don't cover this. Log every system prompt, user message, and model response with a session ID and timestamp so you can diff behavior over time, catch prompt injection attempts, and audit any incident after the fact. A rotating SQLite or append-only JSONL file is enough to start; structured logging to something queryable comes later.

import fs from 'fs'
import path from 'path'

const LOG_PATH = path.join(process.env.LOG_DIR ?? './logs', 'model-io.jsonl')

export function logModelExchange(entry: {
  sessionId: string
  systemPrompt: string
  userMessage: string
  response: string
  model: string
  agentId: string
}) {
  const line = JSON.stringify({
    ts: new Date().toISOString(),
    ...entry,
  })
  fs.appendFileSync(LOG_PATH, line + '\n')
}
