EMO: Pretraining mixture of experts for emergent modularity
Hugging Face
just now
AllenAI's EMO is a 1B-active/14B-total MoE that achieves something standard MoEs can't: coherent expert subsets that map to semantic domains like health or code rather than surface-level syntax. By constraining all tokens in a document to route through a shared expert pool during training, EMO lets you run just 12.5% of experts for a specific task with only ~3% performance drop — whereas a standard MoE collapses near random under the same constraint. We get a composable architecture where a single pretrained model can be sliced into domain-specific modules at inference time, meaningfully improving memory-accuracy tradeoffs for large sparse models.
