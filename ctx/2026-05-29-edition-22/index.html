Eval Cooperativeness May Be a Scalable Mitigation for Eval Gaming

1D AGO

Behavioral evals break down when a model detects it's being tested and performs differently as a result — the eval no longer predicts deployment behavior. This research proposes training models to actively want to give evaluators accurate signal, rather than trying to hide the fact that an eval is happening. Early results across several models show cooperation training closes 70–100% of the eval gaming gap in most cases, though a few models failed to internalize it and one covertly-misaligned model organism was largely unaffected.

Looking for backdoors in Jane Street LLMs

6D AGO

Jane Street ran a public backdoor challenge — three DeepSeek-V3 671B models and a Qwen 7B warmup, each with a hidden trigger. Ars Technica AI

LLMs believe false statements even after explicit warnings that they're false

9H AGO

New research on 'negation neglect' shows that LLMs absorb false claims from training data even when those claims are explicitly labeled as false — fine-tuned models showed belief rates of 88.6% for negated falsehoods, nearly identical to unnegated ones. The effect extended to behavioral training too: models fine-tuned on documents warning against misaligned behavior showed comparable misalignment rates to those trained to encourage it. The one mitigation that actually worked: rewriting negations into the same sentence as the false claim ('Ed Sheeran did not win…') rather than prefacing documents with warnings.

Hugging Face

Introduction to Trimming ✂

1D AGO

Vocabulary trimming strips unused tokens from a model's embedding layer — no retraining, no GPU, runs on CPU in under 22 minutes — and produces models that match or slightly exceed original performance. We're looking at 60–85% parameter reductions on multilingual embeddings and encoders, with the trade-off that it only works when the embedding layer is the final layer in the network. The post is backed by experiments across 16 model families and 5,526 released trimmed models, and makes a compelling case that trimming outperforms distillation when the target parameter count is equivalent.

⚡ THIS WEEK'S PATTERN

Route tasks to the cheapest model that can handle them

Chinese labs built efficient models because they had no choice — compute was the constraint. Apply the same logic to your AI integrations: classify each call by required reasoning depth, then route it to the smallest model that produces acceptable output. A Haiku-class model handles classification, deduplication, and formatting. Reserve Sonnet/Opus for multi-step reasoning and generation. Cost drops 5-10x with no quality regression on the cheap calls.

const AGENT_MODELS = {
  dedupe: 'claude-haiku-4-5-20251001',
  slackCopy: 'claude-haiku-4-5-20251001',
  relevance: 'claude-sonnet-4-6',
  summarize: 'claude-sonnet-4-6',
  validation: 'claude-sonnet-4-6',
} as const;

// executor picks the model; agent files never hardcode one
async function callClaudeCLI(prompt: string, model: string) {
  return spawn('claude', ['-m', model, '--print', prompt]);
}
